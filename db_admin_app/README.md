# DB Admin App - Панель подключения к базам данных

Это Flask-приложение для администрирования баз данных SQLite с поддержкой множественных подключений.

## Новые возможности

### Панель подключения к БД

Приложение теперь работает как полноценная "панель подключения к БД", а не просто админка одной таблицы:

- **Отдельная служебная БД** (`admin.db`) для хранения конфигураций подключений
- **Управление подключениями**: добавление, редактирование, удаление подключений к разным БД
- **Выбор активной БД**: переключение между базами данных через интерфейс
- **Отображение активной БД** в шапке интерфейса

### Резервное копирование

- **Создание бэкапов**: кнопка "Создать бэкап" для каждого подключения
- **Файловые копии**: копирование SQLite-файлов в папку `backups/`
- **История бэкапов**: таблица `backups` в `admin.db` с информацией о копиях

### Статистический дашборд

- **Вкладка "Обзор"**: общая статистика системы
- **Показатели**: количество подключений, бэкапов, пользователей, распределение по ролям
- **Последние действия**: таблица с 10 последними записями из журнала аудита
- **Активная БД**: статистика для выбранной базы данных

### Структура служебной БД

Таблица `connections`:
- `id` - первичный ключ
- `name` - человекочитаемое имя (например, "Учебная БД №1")
- `db_type` - тип СУБД (пока только `sqlite`)
- `db_path` - путь к файлу БД (например, `data/db1.sqlite`)
- `description` - описание подключения
- `read_only` - флаг режима только чтения (0/1)

## Установка и запуск

1. Установите зависимости:
```bash
pip install flask
```

2. Инициализируйте служебную БД:
```bash
python create_admin_db.py
```

3. (Опционально) Создайте тестовую БД:
```bash
python create_test_db.py
```

4. Запустите приложение:
```bash
python app.py
```

5. Откройте браузер: http://localhost:5000

## Использование

1. **Войдите в систему** как администратор (admin/admin)
2. **Перейдите на вкладку "Базы данных"** для управления подключениями
3. **Добавьте новое подключение** или выберите существующее
4. **Сделайте БД активной** кнопкой "Сделать активной"
5. **Работайте с данными** на вкладках "Пользователи", "Роли" и т.д. - операции будут выполняться над выбранной БД

## Архитектура

- **admin.db**: служебная БД с конфигурациями подключений
- **db_web.db**: основная БД (по умолчанию)
- **test.db**: тестовая БД для демонстрации
- Все операции CRUD выполняются над активной БД, определяемой через `session["current_connection_id"]`
